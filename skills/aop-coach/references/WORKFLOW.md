# WORKFLOW.md - AOP 工作流程

基于 AAIF 框架的"探索→构建→验证→学习"循环。

---

## Phase 1: 探索 (Explore)

**目标**：理解问题空间，形成假设

### 步骤

1. **任务分析**
   ```
   - 问题是什么？
   - 业务价值是什么？
   - 成功标准是什么？
   ```

2. **复杂度评估**
   ```
   简单：1个子 Agent
   中等：2-4个子 Agent
   复杂：5+子 Agent
   ```

3. **假设形成**
   ```
   假设 H-001: 如果 [采取行动]，那么 [预期结果]
   验证方法: [如何验证]
   成功标准: [量化指标]
   ```

4. **团队组建**
   ```
   创建 agent team，使用 Sonnet 模型：
   - CoderA：[职责]
   - CoderB：[职责]
   Use Sonnet for each teammate.
   ```

### 输出

- 任务分解清单
- 假设清单
- 子 Agent 分配

---

## Phase 2: 构建 (Build)

**目标**：并行开发，高效交付

### 委派模板

```
任务：[具体描述]

文件范围：
- [文件1] - [职责]

参考代码：
- [现有模块] - [参考什么]

输出要求：
- [交付物1]

边界：
- ✅ 做：[范围]
- ❌ 不做：[范围]

立即开始执行。
```

### 并行执行

```
同时启动多个队友：

- CoderA：数据层
- CoderB：业务层
- CoderC：API层

Wait for all teammates to complete their tasks before proceeding.
```

### 监控进度

```
Shift+Up/Down → 选择队友
Enter → 查看会话状态
```

### 汇总结果

1. 检查每个队友的输出
2. 识别冲突和依赖
3. 协调修复
4. 整合交付

---

## Phase 3: 验证 (Validate)

**目标**：评估解决方案质量

### 代码审查

```
启动 Reviewer Agent：

任务：审查 [模块名] 的代码质量

检查维度：
- 代码风格
- 错误处理
- 安全性
- 性能
- 可维护性

输出要求：
- 问题清单（按严重程度排序）
- 改进建议
```

### 测试验证

```
启动 Tester Agent：

任务：为 [模块名] 编写测试

测试类型：
- 单元测试
- 集成测试

输出要求：
- 测试文件
- 覆盖率报告
```

### 假设验证

```
假设 H-001 验证：
- 预期结果：[...]
- 实际结果：[...]
- 结论：已验证 / 部分验证 / 证伪
```

## Phase 4: 学习 (Learn)

**目标**：捕获经验，持续改进

### 学习日志

```
## 学习日志 [日期]

### 任务
[任务描述]

### 成功经验
- [什么有效]

### 失败教训
- [什么无效]

### 改进建议
- [下次如何做得更好]

### 假设更新
- H-001: 待验证 → 已验证
- H-002: 待验证 → 证伪
```

### 更新知识库

- 记录最佳实践
- 更新技术债务清单
- 记录 API 配置问题

## 规模自适应

### 简单任务

```
流程：分析 → 单 Agent 执行 → 验证 → 学习

时间：几分钟到几小时
子 Agent：1个
```

### 中等任务

```
流程：分析 → 2-4 Agent 并行 → 汇总 → 审查 → 学习

时间：几小时到几天
子 Agent：2-4个
```

### 复杂任务

```
流程：分析 → 多 Agent 并行 → 集成 → 全面测试 → 学习

时间：几天到几周
子 Agent：5+个
```

## 最佳实践（来自 Anthropic）

### 1. 教 Orchestrator 如何委派

每个子 Agent 需要：
- 目标（Objective）
- 输出格式（Output Format）
- 工具和来源（Tools & Sources）
- 清晰边界（Boundaries）

### 2. 规模匹配任务复杂度

- 简单事实查询：1 Agent，3-10 工具调用
- 直接对比：2-4 Agent，10-15 调用
- 复杂研究：10+ Agent，明确分工

### 3. 并行工具调用

- Lead Agent 同时启动 3-5 个子 Agent
- 子 Agent 同时使用 3+ 工具
- 可减少 90% 执行时间

### 4. 先宽后窄

- 先用短、宽泛的查询探索
- 评估可用内容
- 再逐步缩小焦点

## 故障排除

### Agent idle 不执行

1. 切换到队友会话查看错误
2. 检查 API 配置
3. 在队友会话中 `/login`
4. 或重新创建团队，指定模型

### API 403 错误

```
原因：coding_plan_model_not_supported
解决：创建团队时指定 "Use Sonnet for each teammate"
```

### 任务冲突

```
原因：文件范围重叠
解决：使用 Layer-based Assignment
```
