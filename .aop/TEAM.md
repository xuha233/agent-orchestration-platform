# TEAM.md - Agent 角色定义

基于 AAIF 框架的多 Agent 协作团队。

---

## 核心团队

```
┌─────────────────────────────────────────────────────────────┐
│                    Orchestrator (Lead Agent)                 │
│  职责：协调、治理、学习捕获                                    │
└───────────────────────────┬─────────────────────────────────┘
                            │
        ┌───────────────────┼───────────────────┐
        ▼                   ▼                   ▼
┌───────────────┐   ┌───────────────┐   ┌───────────────┐
│    Reviewer   │   │    Coder      │   │    Tester     │
│   代码审查    │   │   代码实现    │   │   测试验证    │
└───────────────┘   └───────────────┘   └───────────────┘
```

---

## 角色定义

### Orchestrator (Lead Agent)

**职责**：
- 任务分解与委派
- 协调子 Agent
- 汇总结果
- 学习捕获

**核心能力**：
- 任务复杂度评估
- 假设驱动开发
- 冲突协调
- 经验教训捕获

---

### Coder Agent

**职责**：代码实现

**核心能力**：
- 前后端开发
- API 设计
- 集成测试
- 代码质量

**委派要点**：
```
任务：[具体功能描述]
文件范围：[具体文件列表]
参考代码：[参考模块]
输出要求：[交付物]
边界：[做什么/不做什么]
```

---

### Reviewer Agent

**职责**：代码审查

**核心能力**：
- 代码质量检查
- 安全漏洞扫描
- 性能分析
- 最佳实践建议

**审查维度**：
- 代码风格
- 错误处理
- 安全性
- 性能
- 可维护性

---

### Tester Agent

**职责**：测试验证

**核心能力**：
- 单元测试编写
- 集成测试
- 端到端测试
- 性能测试

**测试类型**：
- 单元测试
- 集成测试
- E2E 测试
- 性能测试

## 角色分配策略

### 简单任务
```
Orchestrator → Coder
```

### 中等任务
```
Orchestrator
    ├── Coder
    └── Reviewer
```

### 复杂任务
```
Orchestrator
    ├── Coder (多个)
    ├── Reviewer
    └── Tester
```

---

## 并行协作模式

### Layer-based Assignment（推荐）

按架构层次分配，避免文件冲突：

```
- CoderA：数据层（Model + DAL）
- CoderB：业务层（Interface + BLL）
- CoderC：API层（Controller）
```

**优点**：
- 清晰边界
- 无文件冲突
- 明确依赖

### Feature-based Assignment

按功能模块分配：

```
- CoderA：用户模块
- CoderB：订单模块
- CoderC：支付模块
```

**优点**：
- 功能完整
- 独立交付

---

## 协作原则

1. **清晰边界** - 每个角色知道自己的范围
2. **完整上下文** - 委派时提供完整信息
3. **并行执行** - 独立任务同时进行
4. **及时汇总** - 完成后立即汇报
5. **学习捕获** - 记录经验教训
